Перш ніж виставити перший рахунок через KSeF, потрібно до нього увійти. Звучить просто — але
система пропонує кілька різних методів автентифікації, і кожен із них працює по-своєму та
розрахований на різних користувачів.

Profil Zaufany (Довірений Профіль), е-посвідчення, кваліфікований підпис, сертифікат KSeF,
токен — що з цього обрати? І звідки знати, чи твоя програма для виставлення рахунків не
потребує чогось іще?

Якщо ти ФОП (JDG — jednoosobowa działalność gospodarcza, підприємницька діяльність) і
виставляєш кілька рахунків на місяць — відповідь займе хвилину. Якщо обслуговуєш клієнтів
через бухгалтерську контору або інтегруєш KSeF із системою ERP — теж знайдеш тут правильну
відповідь.

---

## Шість методів — коротко

KSeF підтримує два режими доступу: **ручний** (входиш через браузер і працюєш в Aplikacji
Podatnika) та **автоматичний** (програма підключається до KSeF через API без твоєї участі при
кожній операції). Методи автентифікації розподіляються відповідно.

| Метод                              | Режим   | Для кого                                                | Вартість          |
| ---------------------------------- | ------- | ------------------------------------------------------- | ----------------- |
| Profil Zaufany (Довірений Профіль) | браузер | будь-який підприємець                                   | безкоштовно       |
| е-посвідчення (e-dowód)            | браузер | будь-хто з активованим новим посвідченням               | безкоштовно       |
| Кваліфікований підпис / печатка\*  | браузер | фізичні та юридичні особи з кваліфікованим сертифікатом | платно (підписка) |
| Сертифікат KSeF (тип 1 або 2)      | API     | системи ERP, інтегратори                                | безкоштовно       |
| Токен автентифікації               | API     | програми для виставлення рахунків, інтегратори          | безкоштовно       |

\*Підтримку кваліфікованої печатки для юридичних осіб слід перевірити в актуальному FAQ МФ.

Правило просте: входиш вручну через сайт → Profil Zaufany, е-посвідчення або кваліфікований
підпис (підтримку печатки слід перевірити). Твоя програма підключається до KSeF автоматично → їй потрібен сертифікат або
токен.

## Profil Zaufany — для більшості цього достатньо

Profil Zaufany — це безкоштовна державна цифрова ідентичність. Якщо ти колись подавав/подавала
декларацію PIT онлайн, перевіряв/перевіряла е-рецепт або входив/входила до ZUS — швидше за все,
він у тебе вже є.

Якщо ні — оформлення займає кілька хвилин. Це можна зробити через:

- **інтернет-банкінг** — більшість польських банків підтримує цей процес без виходу з дому,
- **застосунок mObywatel** на телефоні,
- **ePUAP** (pz.gov.pl),
- або особисто в органі податкової служби.

У KSeF Profil Zaufany дає повний доступ до Aplikacji Podatnika за адресою
[ap.ksef.mf.gov.pl](https://ap.ksef.mf.gov.pl/). Там можна виставляти рахунки через форму,
переглядати документи від контрагентів і керувати доступами для інших осіб — наприклад, для
бухгалтерської контори.

Є одне важливе обмеження: Profil Zaufany призначений лише для **ручного входу**. Твоя програма
для виставлення рахунків — Fakturownia, inFakt, wFirma чи будь-яка інша — підключається до KSeF
через API, і там Profil Zaufany не працює. Програмі потрібен токен або сертифікат. Але не треба
розбиратися з цим самостійно — під час першого налаштування програма сама проведе тебе через
цей крок.

<Info>
  Для ФОП, який виставляє кілька рахунків на місяць, Profil Zaufany цілком достатній. Якщо
  користуєшся комерційною програмою, налаштування токена — одноразове завдання, яке зазвичай займає
  кілька хвилин.
</Info>

## Е-посвідчення — якщо воно вже активоване

Нові польські посвідчення особи мають вбудований чіп і підтримують електронний вхід. Якщо твоє
е-посвідчення (e-dowód) активоване для підпису, ним можна входити до KSeF так само, як і через
Profil Zaufany. Активація потребує одноразового візиту до органу влади.

На практиці е-посвідчення обирають рідше — Profil Zaufany зручніший, бо не потребує зчитувача
карток. Але якщо у тебе вже є активоване е-посвідчення і ти користуєшся ним в інших цілях —
воно чудово підійде і для KSeF.

## Кваліфікований підпис і кваліфікована печатка — для компаній

**Кваліфікований електронний підпис** — це комерційний аналог власноручного підпису, юридично
дійсний на рівні Євросоюзу (стандарт eIDAS). Його видають комерційні постачальники — зокрема
Certum, Sigillum та EuroCert — за річною або багаторічною підпискою. Він коштує грошей.

У контексті KSeF кваліфікований підпис дає такий самий доступ через браузер, що й Profil Zaufany
— повний доступ до Aplikacji Podatnika. Кому варто його використовувати? Насамперед тим, хто
вже має підпис для інших цілей: e-KRS, публічні тендери, підписання договорів. Якщо ти його не
маєш і не потребуєш для інших цілей — Profil Zaufany дешевший і настільки ж ефективний.

Стандарт eIDAS розрізняє кваліфікований підпис (інструмент фізичної особи) та **кваліфіковану
печатку** (інструмент юридичної особи, наприклад компанії). Чи підтримує KSeF кваліфіковану
печатку як окремий метод входу і в якому обсязі — завжди перевіряй в актуальному FAQ МФ або в
Посібнику Aplikacji Podatnika KSeF 2.0.

<Info>
  Актуальний перелік підтримуваних методів автентифікації — зокрема можливу підтримку кваліфікованої
  печатки для юридичних осіб — завжди перевіряй в офіційному{" "}
  <a href="https://ksef.podatki.gov.pl/pytania-i-odpowiedzi-ksef-20">FAQ МФ</a>. Це авторитетне
  джерело.
</Info>

## Сертифікат KSeF — два типи, два призначення

Сертифікат KSeF — це електронне посвідчення особи для доступу через API: для програм і систем,
які спілкуються з KSeF автоматично, без участі людини при кожній операції. Отримують його в
системі Міністерства фінансів.

Є два типи, які суттєво відрізняються.

**Сертифікат типу 1** видається безпосередньо платнику податків — власнику номера NIP. Він має
найширші повноваження в рамках цього NIP. Використовується, коли власна система компанії
інтегрується з KSeF від свого імені — наприклад, коли компанія підключає свій ERP безпосередньо
до KSeF як себе, а не через посередника.

**Сертифікат типу 2** призначений для суб'єктів, що діють від імені платника: бухгалтерських
контор, систем, що обслуговують багатьох клієнтів, зовнішніх інтеграторів. Ключове правило:
сертифікат типу 2 завжди діє в контексті конкретного уповноваження. Без дійсної авторизації від
платника — наприклад ZAW-FA або дозволу, наданого в панелі Aplikacji Podatnika — сертифікат
типу 2 просто не спрацює.

Коротко: **тип 1 — коли система діє під власним NIP компанії, тип 2 — коли діє від імені чужого
NIP.**

<Warning>
  Деталі процесу отримання сертифіката можуть змінюватися разом із оновленнями системи МФ. Перш ніж
  розпочати, перевір актуальну інструкцію в Посібнику Aplikacji Podatnika KSeF 2.0, доступному на
  ksef.podatki.gov.pl.
</Warning>

## Токен — найпростіший доступ через API

Токен автентифікації — це рядок символів, ключ доступу, який ти самостійно генеруєш в Aplikacji
Podatnika KSeF. Він не вимагає керування інфраструктурою сертифікатів. Генеруєш у застосунку,
вводиш у налаштуваннях своєї програми — і готово.

Токен має визначений термін дії та обсяг повноважень — можна обмежити його, наприклад, лише
виставленням рахунків або лише читанням. У будь-який момент його можна відкликати з Aplikacji
Podatnika одним кліком. Це важливо: якщо зміниш постачальника програми або щось піде не так —
миттєве позбавлення доступу нічого не коштує.

Коли обрати токен замість сертифіката? Токени зручніші для інтеграторів і менших систем, бо не
потребують налаштування PKI-інфраструктури. Для складних впроваджень, що обслуговують багатьох
платників одночасно, сертифікат може бути кращим рішенням — більш надійні облікові дані і ширший
контроль.

<Tip>
  Якщо твоя програма просить «налаштувати підключення до KSeF» або «ввести токен» — увійди до{" "}
  <a href="https://ap.ksef.mf.gov.pl/">Aplikacji Podatnika</a>, знайди розділ керування токенами і
  згенеруй новий. Програма зазвичай веде через цей крок покроково.
</Tip>

## ZAW-FA — коли хтось діє від твого імені

Якщо ти користуєшся бухгалтерською конторою або хочеш надати комусь іншому доступ до свого KSeF
— тобі потрібне уповноваження ZAW-FA.

ZAW-FA — це офіційна форма авторизації. Вона дозволяє уповноважити конкретну особу або суб'єкт
(наприклад, свого бухгалтера або бухгалтерську контору) виставляти та читати рахунки від твого
імені. Подати уповноваження можна кількома способами:

- **особисто в органі податкової служби**,
- **через e-Urząd Skarbowy** ([podatki.gov.pl](https://www.podatki.gov.pl/e-urzad-skarbowy/)) — в електронній формі,
- **безпосередньо в Aplikacji Podatnika KSeF** — через панель доступів, не виходячи із системи.

Обсяг доступу можна обмежити трьома варіантами: лише читання рахунків, лише виставлення або
повний доступ. Варто узгодити обсяг із бухгалтером перед наданням дозволів — краще підібрати його
під реальні потреби, ніж давати ширші права, ніж потрібно.

Добра новина для ФОП: якщо ти користуєшся бухгалтерською конторою, вона зазвичай сама попросить
тебе про відповідне уповноваження і проведе через весь процес. Не треба знати деталей —
запитай свого бухгалтера.

Для бухгалтерських контор та їхніх клієнтів це ключове правило: ZAW-FA від кожного клієнта є
обов'язковою умовою перед запуском будь-якої інтеграції через API. Сертифікат типу 2 без
активного уповноваження просто не спрацює.

## Яке рішення підходить тобі?

Таблиця нижче — практична підказка, не юридична консультація. Завжди перевіряй актуальний стан у
[FAQ МФ](https://ksef.podatki.gov.pl/pytania-i-odpowiedzi-ksef-20).

| Твоя ситуація                                            | Рекомендований метод                                                           |
| -------------------------------------------------------- | ------------------------------------------------------------------------------ |
| ФОП, виставляєш рахунки вручну в Aplikacji Podatnika     | **Profil Zaufany**                                                             |
| Користуєшся програмою (Fakturownia, inFakt, wFirma тощо) | **Profil Zaufany** для ручного входу + **токен**, згенерований для програми    |
| Компанія, власна система ERP підключається до KSeF       | **Сертифікат KSeF тип 1** або токен                                            |
| Бухгалтерська контора, що обслуговує багатьох клієнтів   | **ZAW-FA** від кожного клієнта + **сертифікат тип 2** (або токен)              |
| Розробник, що інтегрує KSeF із системою компанії         | **Токен** (простий старт) або **сертифікат тип 1/2** (продакшн)                |
| Маєш кваліфікований підпис для інших цілей               | Можеш використати замість Profil Zaufany — дає ідентичний доступ через браузер |

## e-mikrofirma — якщо хочеш максимальної простоти

Якщо виставляєш рахунки зрідка і воліш зосередитися на роботі, а не на налаштуванні доступу —
зверни увагу на **e-mikrofirmę**. Це безкоштовний веб-застосунок МФ для найменших підприємців,
що охоплює базове виставлення рахунків і податкові розрахунки.

e-mikrofirma підтримує KSeF в обмеженому обсязі. Перш ніж обрати цей інструмент, перевір
актуальний перелік функцій на сайті МФ — обсяг підтримки KSeF може змінюватися разом із
оновленнями застосунку. Для тих, хто виставляє рахунки зрідка, це може бути достатнім і
найпростішим варіантом.

Більше про підготовку до KSeF як ФОП читай у статті
[KSeF з 1 квітня 2026 — що змінюється](/uk/blog/ksef-z-1-kvitnya-2026).

## Перш ніж рахунок потрапить до системи — перевір його

Вибір методу входу — це лише половина справи. Перш ніж твій рахунок потрапить до KSeF —
переконайся, що він правильний. Рахунок, відхилений системою, юридично не існує — навіть якщо
клієнт отримав його поштою у форматі PDF.

Завантаж XML-файл на **[ksefuj.to](/)** — безкоштовний валідатор, що відповідає офіційним вимогам
МФ. Він повністю працює в браузері: твій файл не виходить за межі браузера. Результат — за
кілька секунд.

Маєш більше питань про вхід, сертифікати та доступи в KSeF? Відповіді на найпоширеніші з них
знайдеш у нашому розділі [FAQ](/faq).
