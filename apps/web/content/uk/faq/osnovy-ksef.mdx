## Основи KSeF

<Question q="Що таке KSeF?">
  KSeF (Krajowy System e-Faktur) — це платформа Міністерства фінансів Польщі для виставлення та
  отримання структурованих рахунків-фактур. Кожен рахунок B2B має бути надісланий до KSeF у форматі
  XML і отримати офіційний номер перш ніж потрапити до покупця. Система діє як центральний реєстр
  податкових рахунків, що ведеться державою.
</Question>

<Question q="З якого часу KSeF є обов'язковим?">
  Для великих підприємців (продажі понад 200 млн злотих на рік) KSeF є обов'язковим з 1 лютого 2026
  року. Для решти платників ПДВ — з 1 квітня 2026 року. Суб'єкти, звільнені від ПДВ, приєднуються до
  системи з 1 січня 2027 року.
</Question>

<Question q="Кого стосується обов'язок KSeF?">
  KSeF стосується всіх активних платників ПДВ, зареєстрованих у Польщі. Обов'язок охоплює рахунки
  B2B та рахунки для фізичних осіб, виставлені поза касовим апаратом. Суб'єкти, звільнені від ПДВ,
  приєднуються з 1 січня 2027 року.
</Question>

<Question q="Чи стосується KSeF рахунків для фізичних осіб (B2C)?">
  Рахунки для споживачів (B2C) загалом підпадають під KSeF, але з важливим винятком: рахунки,
  виставлені через касовий апарат, виключені з системи. Якщо ви виставляєте рахунок поза касою —
  наприклад, у програмі для виставлення рахунків — він має пройти через KSeF. На практиці більшість
  роздрібних рахунків, виданих через касу, залишаються поза системою.
</Question>

<Question q="Чи стосується KSeF чеків з NIP?">
  Касові чеки з NIP покупця, виставлені через касовий апарат, вважаються спрощеними рахунками і не
  входять до KSeF. Ці чеки зберігають повну юридичну силу як документ про покупку. Тільки звичайні
  рахунки ПДВ, виставлені поза касою, підпадають під обов'язок KSeF.
</Question>

<Question q="Що таке FA(3)?">
  FA(3) — це діюча схема XML для структурованих рахунків-фактур, опублікована Міністерством фінансів
  Польщі. Цифра 3 означає третю версію формату. Усі рахунки, що надсилаються до KSeF, повинні
  відповідати цій схемі — будь-яке відхилення призводить до відхилення рахунку системою.
</Question>

<Question q="У чому різниця між FA(2) та FA(3)?">
  FA(2) — це попередня версія схеми, що застосовувалась у KSeF 1.0 (до кінця 2025 року). FA(3)
  запровадила новий простір імен XML, оновлені поля та розширені вимоги до анотацій. Рахунки у
  форматі FA(2) не приймаються KSeF 2.0 — якщо ваша програма генерувала FA(2), переконайтесь, що
  постачальник випустив оновлення до FA(3).
</Question>

<Question q="Чи можу я надсилати рахунки в PDF?">
  Ні — PDF не приймається KSeF як податковий документ. Система вимагає структурованого XML-файлу
  відповідно до FA(3). Ви можете надсилати покупцю візуалізацію рахунку у форматі PDF для зручності,
  але юридичною версією документа є XML-файл, зареєстрований у KSeF. Хороша програма для виставлення
  рахунків генерує обидва: XML для KSeF і PDF для покупця.
</Question>

<Question q="Що таке номер KSeF?">
  Номер KSeF — це унікальний ідентифікатор рахунку, що присвоюється системою МФ після успішної
  реєстрації документа. Цей номер потрібен при виставленні коригувального рахунку — коригування
  повинно посилатися на номер KSeF оригінального рахунку, інакше буде відхилено. Якщо оригінальний
  рахунок був виставлений поза KSeF (до його впровадження), замість номера KSeF використовується
  мітка NrKSeFN.
</Question>

<Question q="Що таке UPO?">
  UPO (Urzędowe Poświadczenie Odbioru — Офіційне підтвердження отримання) — це документ, що
  підтверджує прийняття рахунку системою KSeF. Містить номер KSeF, дату прийняття та цифровий підпис
  системи. UPO доводить, що рахунок був правильно зареєстрований — його варто зберігати разом з
  XML-файлом рахунку.
</Question>
