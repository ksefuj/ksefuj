## Доступ та повноваження

<Question q="Як увійти до KSeF?">
  Увійдіть до KSeF через Aplikację Podatnika (ap.ksef.mf.gov.pl) одним із способів: Довірений
  профіль (Profil Zaufany), е-посвідчення, кваліфікований підпис або сертифікат KSeF. Довірений
  профіль — найзручніший для більшості підприємців, його можна отримати безкоштовно через
  онлайн-банкінг або в органах влади. Доступ через API (для програм) потребує сертифіката або
  токена.
</Question>

<Question q="Що таке ZAW-FA?">
  ZAW-FA — це форма уповноваження для використання KSeF від імені платника податків. Її подають у
  податковій інспекції або електронно через e-Urząd Skarbowy, і вона дозволяє уповноважити іншу
  особу — наприклад, бухгалтера або бухгалтерську фірму — виставляти та отримувати рахунки в KSeF.
  Уповноваження може бути обмеженим: лише читання, лише виставлення або повний доступ.
</Question>

<Question q="Як уповноважити бухгалтера для KSeF?">
  Ви можете уповноважити бухгалтера, подавши форму ZAW-FA в податковій інспекції або безпосередньо в
  Aplikacji Podatnika KSeF через панель повноважень. Уповноваження визначає, чи може бухгалтер лише
  переглядати рахунки, чи також їх виставляти. Бухгалтерська фірма зазвичай сама просить клієнтів
  про відповідне уповноваження — запитайте свого бухгалтера.
</Question>

<Question q="Що таке сертифікат KSeF?">
  Сертифікат KSeF — це електронне посвідчення особи, що використовується для автентифікації в API
  KSeF. Завантажується з системи МФ і дозволяє програмам та інтеграторам входити до KSeF без
  Довіреного профілю. Розрізняють два типи сертифікатів: для платника податків (тип 1) і для
  уповноваженого суб'єкта (тип 2).
</Question>

<Question q="У чому різниця між сертифікатом типу 1 і типу 2?">
  Сертифікат типу 1 видається безпосередньо платнику податків (власнику NIP) і має найширші
  повноваження в KSeF. Сертифікат типу 2 призначений для суб'єктів, що діють від імені платника —
  наприклад, бухгалтерських фірм або ERP-систем, що обслуговують кількох клієнтів. Тип 2 завжди діє
  в межах конкретного уповноваження, яке має існувати в KSeF.
</Question>

<Question q="Що таке токен автентифікації?">
  Токен автентифікації — це рядок символів, що генерується в Aplikacji Podatnika KSeF і
  використовується замість сертифіката для автентифікації в API. Токени зручніші для інтеграторів і
  менших систем — не потребують управління PKI-сертифікатами. Токен має визначений термін дії та
  обсяг повноважень і може бути відкликаний у будь-який момент з Aplikacji Podatnika.
</Question>
