Walidator **ksefuj** sprawdza pliki XML pod kątem zgodności ze schematem FA(3) Ministerstwa Finansów.
Działa dwuetapowo:

1. **Walidacja XSD** — sprawdzenie struktury XML względem oficjalnego schematu
2. **Walidacja semantyczna** — sprawdzenie reguł biznesowych, których XSD nie obejmuje

## Jak używać walidatora

### Przeglądarka (online)

Wejdź na [ksefuj.to](https://ksefuj.to) i przeciągnij plik XML do strefy walidacji. Wyniki pojawią
się natychmiast — bez przesyłania danych na serwer.

### CLI

```bash
npx @ksefuj/validator faktura.xml
```

### npm

```typescript
import { validate } from "@ksefuj/validator";

const result = await validate(xmlString);
console.log(result.errors); // błędy XSD
console.log(result.warnings); // ostrzeżenia semantyczne
```

## Interpretacja wyników

<FieldTable>
  <Field name="errors" type="ValidationIssue[]">
    Błędy blokujące — faktura nie zostanie przyjęta przez KSeF
  </Field>
  <Field name="warnings" type="ValidationIssue[]">
    Ostrzeżenia — faktura może być przyjęta, ale warto poprawić
  </Field>
  <Field name="isValid" type="boolean">
    true jeśli brak błędów (ostrzeżenia nie wpływają na isValid)
  </Field>
</FieldTable>

<Warning>
  Brak błędów w walidatorze ksefuj **nie gwarantuje przyjęcia** faktury przez KSeF — system MF może
  stosować dodatkowe reguły biznesowe nieujęte w schemacie FA(3).
</Warning>
